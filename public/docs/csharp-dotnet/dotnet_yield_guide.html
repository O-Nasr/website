yield Keyword — Complete Guide

1. What is yield?

yield enables custom stateful iteration over collections without creating temporary collections. It moves control back and forth between the caller and the method — remembering exactly where it left off each time.
Simple rule: Instead of collecting everything first and returning it all at once — yield returns ONE item at a time, on demand.
// Without yield — collects everything first
static IEnumerable<int> FilterWithoutYield()
{
    List<int> temp = new List<int>(); // ❌ extra temporary collection
    foreach (int i in MyList)
    {
        if (i > 3) temp.Add(i);
    }
    return temp; // returns everything at once
}

// With yield — returns one item at a time
static IEnumerable<int> FilterWithYield()
{
    foreach (int i in MyList)
    {
        if (i > 3) yield return i; // return ONE item → pause → wait for caller
    }
}

2. How yield Works — Step by Step

When the caller iterates — control flows back and forth between the caller and the method:

static List<int> MyList = new List<int> { 1, 2, 3, 4, 5 };

static void Main()
{
    foreach (int i in FilterWithYield())
    {
        Console.WriteLine(i); // prints 4, then 5
    }
}

// What happens internally:
// Step 1 → Main calls FilterWithYield() — starts iterating
// Step 2 → Function loops: 1 → no, 2 → no, 3 → no, 4 → YES
//           yield return 4 → sends 4 to Main → PAUSES
// Step 3 → Main prints 4 → asks for next item
// Step 4 → Function RESUMES from where it left off → 5 → YES
//           yield return 5 → sends 5 to Main → PAUSES
// Step 5 → Main prints 5 → asks for next item
// Step 6 → No more items → function ends → loop stops

⚠️ Important: Think of yield like a ping pong game — control bounces back and forth between the caller and the method. The method remembers its position between each bounce.

4. Scenario 2 — Stateful Iteration

yield preserves local variables between calls. This allows running totals, counters, and any state to persist across iterations.
// Running total — state preserved between yields
static IEnumerable<int> RunningTotal()
{
    int runningTotal = 0; // this variable STAYS alive between yields

    foreach (int i in MyList)
    {
        runningTotal = runningTotal + i; // update state
        yield return runningTotal;       // send current total → pause
    }
}

// Output when called:
// 1   (0+1)
// 3   (1+2)
// 6   (3+3)
// 10  (6+4)
// 15  (10+5)

// Why is this special?
// Normally when a method returns — all local variables are DESTROYED

// With yield — method is PAUSED not ended — variables stay alive

5. yield return vs yield break

yield return i;  // return this value and PAUSE — continue on next iteration

yield break;     // stop iteration completely — like a return statement

// Example of yield break
static IEnumerable<int> StopAtThree()
{
    foreach (int i in MyList)
    {
        if (i > 3)
            yield break; // stop everything here

        yield return i;
    }
}
// Output: 1, 2, 3 — stops before 4

7. yield is Inherently Lazy

yield connects directly to Lazy Loading — the method only executes when the caller actually iterates. If the caller stops early, the remaining items are never processed.
// Only processes items until caller stops
foreach (int i in FilterWithYield())
{
    Console.WriteLine(i);
    if (i == 4) break; // caller stops here — 5 is NEVER processed
}
// Method only ran up to 4 — never touched 5 — efficient ✅

8. Interview Questions

Q: What is the yield keyword used for?

🎯 Interview Answer: yield enables custom stateful iteration over collections without creating temporary collections. When yield return is hit, the current value is sent to the caller and the method pauses — preserving all local variables. When the caller asks for the next value, the method resumes exactly where it left off. This is useful for filtering collections on the fly without building a temporary list, and for maintaining running state between iterations like a running total.

Q: What is the difference between yield return and yield break?

🎯 Interview Answer: yield return sends the current value to the caller and pauses the method — execution continues from that point on the next iteration request. yield break stops the iteration completely — similar to a return statement but for iterator methods. After yield break no more values are returned to the caller.

Q: What does the compiler do with yield?

🎯 Interview Answer: The compiler transforms any method containing yield into a state machine class behind the scenes. This state machine implements IEnumerator and tracks the current position, local variable values, and the current state of execution. This is why local variables survive between yield return calls — they are actually stored as fields in the generated state machine class.
