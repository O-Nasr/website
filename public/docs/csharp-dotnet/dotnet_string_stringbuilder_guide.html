String vs StringBuilder — Complete Guide

1. Why String is Immutable

In C#, strings are immutable — every time you modify or concatenate a string, a brand new string object is created in memory. The old one becomes garbage for the GC to collect.
string result = "";
result = result + "A"; // allocates NEW string "A" at location #2 — location #1 abandoned
result = result + "B"; // allocates NEW string "AB" at location #3 — location #2 abandoned
result = result + "C"; // allocates NEW string "ABC" at location #4 — location #3 abandoned

In a loop of 10,000 iterations:

  • 10,000 new string allocations in memory
  • 10,000 old strings abandoned for GC to clean
  • Massive memory pressure + slow performance 😱

2. StringBuilder — The Solution

StringBuilder works completely differently. It maintains a single mutable buffer in memory and keeps appending to it directly — no new allocations on every change.
StringBuilder sb = new StringBuilder();

for (int i = 0; i < 10000; i++)
{
    sb.Append(i.ToString()); // modifies the SAME buffer — no new allocation
}

string result = sb.ToString(); // convert to string ONCE at the end

What happens in memory:

  • Step 1 → sb created → ONE buffer allocated
  • Step 2 → sb.Append() → writes to SAME buffer
  • Step 10000 → sb.Append() → STILL the same buffer
  • Final → sb.ToString() → ONE string created at the end

4. Common StringBuilder Methods

StringBuilder sb = new StringBuilder();

sb.Append("Hello");           // add text at the end
sb.AppendLine("Hello");       // add text + new line
sb.Insert(0, "Start: ");      // insert at specific position
sb.Replace("Hello", "Hi");    // find and replace
sb.Remove(0, 5);              // remove characters
sb.Clear();                   // clear everything
sb.Length;                    // current length

string result = sb.ToString(); // convert to string when done

6. String Interning — Bonus Point

.NET has a String Intern Pool. When you write string literals, .NET stores them only once in memory and multiple variables point to the same location.
string a = "Hello";
string b = "Hello";
Console.WriteLine(object.ReferenceEquals(a, b)); // true — same memory location!

// But dynamically built strings are NOT interned
string c = new string(new char[]{'H','e','l','l','o'}); // NOT interned
Console.WriteLine(object.ReferenceEquals(a, c)); // false — different locations

7. Interview Questions

Q: Why is string concatenation in a loop bad?

🎯 Interview Answer: In C#, strings are immutable — every concatenation creates a brand new string object in memory and the old one becomes garbage for the GC to collect. In a loop with thousands of iterations this causes massive memory pressure and performance degradation as thousands of string objects are created and abandoned.

Q: What is StringBuilder and when should you use it?

🎯 Interview Answer: StringBuilder maintains a single mutable buffer in memory and appends to it directly without creating new objects. It should be used whenever strings are concatenated multiple times — especially inside loops. StringBuilder can be up to 100x faster than string concatenation in a loop because it only creates one string allocation instead of thousands.

Q: What is string immutability?

🎯 Interview Answer: String immutability means that once a string object is created in memory it cannot be changed. Any operation that appears to modify a string — like concatenation or Replace — actually creates a brand new string object. This is why string is thread-safe by design but also why repeated modification is expensive.
