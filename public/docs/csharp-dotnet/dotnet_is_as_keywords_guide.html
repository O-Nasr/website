2. IS Keyword — Type Checking

IS simply asks a yes/no question: "Is this object of this type?"

object obj = "Hello";

if (obj is string)
{
    Console.WriteLine("Yes! obj is a string"); // ✅ prints this
}

// Works with inheritance too
class Animal { }
class Dog : Animal { }

Dog d = new Dog();
Console.WriteLine(d is Dog);    // true
Console.WriteLine(d is Animal); // true — Dog inherits from Animal
Console.WriteLine(d is string); // false

3. AS Keyword — Safe Type Conversion

AS tries to convert the object to the given type. If it fails it returns null instead of throwing an exception.
// Successful conversion
object obj = "Hello";
string str = obj as string;
Console.WriteLine(str); // "Hello" ✅

// Failed conversion — returns null NOT exception
object obj = 42;
string str = obj as string;
Console.WriteLine(str == null); // true — safe, no crash ✅

// vs direct cast — DANGEROUS
object obj = 42;
string s = (string)obj; // 💥 InvalidCastException thrown!

⚠️ Important: AS never throws an exception on failed conversion. Direct cast (string)obj throws InvalidCastException. Always prefer AS for uncertain conversions.

4. AS Only Works With Reference Types

object obj = 42;

int i = obj as int;    // ❌ COMPILE ERROR — AS doesn't work with value types

// Fix — use IS with pattern matching for value types

if (obj is int i)      // ✅ pattern matching works fine
{
    Console.WriteLine(i);
}

5. Modern C# — Pattern Matching (C# 7+)

Since C# 7, you can combine IS with variable assignment in one line. This is the preferred modern approach.
// Old way — checks type twice internally
if (obj is string)
{
    string s = (string)obj; // two operations
    Console.WriteLine(s);
}

// Modern way — pattern matching
if (obj is string s) // checks type AND assigns in one shot ✅
{
    Console.WriteLine(s); // s is already typed as string here
}

// Switch pattern matching
switch (obj)
{
    case string s:
        Console.WriteLine("String: " + s);
        break;
    case int i:
        Console.WriteLine("Int: " + i);
        break;
    case null:
        Console.WriteLine("Null");
        break;
}

7. Interview Questions

Q: What is the difference between IS and AS?

🎯 Interview Answer: The IS keyword checks if an object is compatible with a given type and returns true or false without performing any conversion. The AS keyword attempts to convert an object to a given type — if successful it returns the converted object, if it fails it returns null instead of throwing an exception. This makes AS safer than a direct cast for uncertain conversions.

Q: What is pattern matching in C#?

🎯 Interview Answer: Pattern matching introduced in C# 7 extends the IS keyword to combine type checking and variable assignment in a single statement. Instead of checking is string then casting separately, you write if (obj is string s) which checks the type and assigns the typed variable in one operation. This eliminates the double-check overhead and produces cleaner more readable code.

Q: Why can't AS be used with value types?

🎯 Interview Answer: AS returns null when conversion fails, but value types like int and bool cannot be null. Therefore AS is only valid for reference types and nullable value types. For value types, use pattern matching with IS which handles the type check and conversion safely without requiring nullability.
