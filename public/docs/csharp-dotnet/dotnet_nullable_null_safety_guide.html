Nullable Types and Null Safety — Complete Guide

1. Value Types vs Reference Types and Null

int    age  = null;     // ❌ compile error — value types cannot be null
string name = null;     // ✅ fine — reference types can be null

// Stack — value types: int, bool, double, char → always has a value
// Heap  — reference types: string, class, array → can point to null

2. Nullable Value Types with ?

int?      age      = null; // ✅ nullable int
bool?     isActive = null; // ✅ nullable bool
double?   salary   = null; // ✅ nullable double
DateTime? dob      = null; // ✅ nullable DateTime

// int? is shorthand for Nullable<int>
int? age = 25;
Console.WriteLine(age.HasValue); // true
Console.WriteLine(age.Value);    // 25

int? salary = null;
Console.WriteLine(salary.HasValue); // false
Console.WriteLine(salary.Value);    // ❌ InvalidOperationException!

// Always check HasValue before accessing Value
if (salary.HasValue)
    Console.WriteLine("Salary: " + salary.Value);

3. The 4 Null Safety Operators

1. ?? — Null Coalescing

Returns left side if not null, otherwise returns right side.
string city   = null;
string result = city ?? "Cairo"; // city is null → returns "Cairo"

// Chaining — first non-null wins
string display = firstName ?? lastName ?? username ?? "Guest";

// Real world
string email = reader["Email"] as string ?? "no-email@default.com";

2. ??= — Null Coalescing Assignment (C# 8+)

Assigns right side value ONLY if left side is null.
string name = null;
name ??= "Default Name"; // name is null → assign "Default Name"
Console.WriteLine(name); // Default Name

string city = "Cairo";
city ??= "Alexandria"; // city is NOT null → unchanged
Console.WriteLine(city); // Cairo

3. ?. — Null Conditional Operator

Accesses a property or calls a method safely — returns null instead of throwing NullReferenceException.
string name = null;

// ❌ Old way — crashes
int length = name.Length; // 💥 NullReferenceException

// ✅ New way — safe
int? length = name?.Length; // null instead of crash

// Chaining — multiple levels deep
string city = order?.Customer?.Address?.City;
// If ANY link is null → returns null safely

// Combining ?. with ?? — best pattern
string city = order?.Customer?.Address?.City ?? "Unknown";

4. ! — Null Forgiving Operator (C# 8+)

Tells compiler you are certain the value is not null. Suppresses nullable warnings. Use sparingly.
string? name = GetName(); // might return null

// ⚠️ Compiler warns — name might be null
int length = name.Length;

// You KNOW it's not null — tell compiler to trust you
int length = name!.Length; // ! suppresses warning — you take responsibility

4. Nullable Reference Types (C# 8+)

// Enable in .csproj
<Nullable>enable</Nullable>

// Before — all reference types nullable without warning
string name = null;  // no warning — risky

// After enabling — compiler tracks nullability
string  name = null;  // ⚠️ WARNING — non-nullable shouldn't be null
string? city = null;  // ✅ explicitly marked as nullable

// Compiler warns on unsafe access
string? name = GetName();
Console.WriteLine(name.Length); // ⚠️ WARNING — name might be null

// Fix options:
if (name != null) Console.WriteLine(name.Length);  // null check
Console.WriteLine(name?.Length);                   // null conditional
Console.WriteLine(name!.Length);                   // null forgiving

5. Guard Clauses — Null Checking at Entry

public void ProcessOrder(Order order)
{
    // Classic guard clause
    if (order == null)
        throw new ArgumentNullException(nameof(order));

    // Modern C# 10+ way
    ArgumentNullException.ThrowIfNull(order);

    // Safe to use order here
    Console.WriteLine(order.OrderNumber);
}

6. Interview Questions

Q: What is the difference between ? for value types and reference types?

🎯 Interview Answer: For value types, the ? syntax creates a Nullable<T> struct which wraps the value type and adds HasValue and Value properties. This allows value types like int and bool to represent the absence of a value. For reference types, adding ? in C# 8+ with nullable reference types enabled tells the compiler to track whether that variable might be null and warn when it is accessed unsafely.

Q: What is the difference between ?? and ?.?

🎯 Interview Answer: The ?? operator is the null coalescing operator — it returns the left operand if it is not null, otherwise returns the right operand as a fallback value. The ?. operator is the null conditional operator — it accesses a member only if the object is not null, returning null instead of throwing NullReferenceException if it is null. They are often combined: order?.Customer?.City ?? 'Unknown' safely navigates a chain and provides a fallback.
