LINQ + Deferred Execution — Complete Guide

1. What is LINQ?

LINQ provides a unified way to query collections, databases, and other data sources using either method syntax or query syntax — both produce identical results.
// Method syntax — preferred
var result = numbers.Where(n => n > 3).Select(n => n * 2).ToList();

// Query syntax — same result
var result = (from n in numbers where n > 3 select n * 2).ToList();

2. Most Common LINQ Methods

var evens    = numbers.Where(n => n % 2 == 0);           // filter
var doubled  = numbers.Select(n => n * 2);               // transform
var sorted   = numbers.OrderBy(n => n);                  // sort asc
var first    = numbers.First(n => n > 5);                // first match — throws if none
var firstOr  = numbers.FirstOrDefault(n => n > 99);      // first or default — no exception
var hasEvens = numbers.Any(n => n % 2 == 0);             // true/false check
var allPos   = numbers.All(n => n > 0);                  // all match?
var count    = numbers.Count(n => n % 2 == 0);           // count matches
var sum      = numbers.Sum();                            // sum all
var page1    = numbers.Skip(0).Take(3);                  // pagination
var unique   = numbers.Distinct();                       // remove duplicates
var list     = numbers.Where(n => n > 5).ToList();       // execute and store

3. GroupBy — Common Interview Question

var grouped = employees
    .GroupBy(e => e.Department)
    .Select(g => new
    {
        Department = g.Key,
        Count      = g.Count(),
        AvgSalary  = g.Average(e => e.Salary)
    });

// Output:
// IT: 3 employees, avg salary 5500
// HR: 2 employees, avg salary 5000

4. Deferred Execution — Most Important Concept

When you write a LINQ query it does NOT execute immediately. It only executes when you actually iterate over the results. The query is like a recipe — writing it doesn't cook the food. Iterating cooks it.
var query = numbers.Where(n => n > 3); // NO execution yet

Console.WriteLine("Before iteration");

foreach (int n in query) // execution happens HERE
{
    Console.WriteLine(n);
}

Dangerous — List Changes After Query Defined

var query = numbers.Where(n => n > 3);

numbers.Add(10); // modify AFTER query defined
numbers.Add(11);

foreach (int n in query) // runs against MODIFIED list
    Console.WriteLine(n);
// Output: 4, 5, 10, 11 — changes included!

Dangerous — Multiple Enumeration

var query = numbers.Where(n => n > 3);

int count = query.Count();  // Execution #1 — full pass through all elements
foreach (int n in query)    // Execution #2 — full pass AGAIN
    Console.WriteLine(n);

// With Entity Framework: TWO separate SQL queries fired! 😱

Fix — ToList() Forces Immediate Execution

List<int> result = numbers.Where(n => n > 3).ToList(); // execute once

numbers.Add(10); // ignored — result already stored

int count = result.Count;   // reads list — no re-execution
foreach (int n in result)   // reads list — no re-execution

5. IEnumerable vs IQueryable

// IEnumerable — filters IN MEMORY
IEnumerable<Employee> result = dbContext.Employees
    .ToList()                      // pulls ALL rows from DB first
    .Where(e => e.Salary > 5000);  // filters in C# memory
// SQL: SELECT * FROM Employees  ← transfers everything!

// IQueryable — filters IN DATABASE
IQueryable<Employee> result = dbContext.Employees
    .Where(e => e.Salary > 5000);  // translated to SQL WHERE clause
// SQL: SELECT * FROM Employees WHERE Salary > 5000  ← efficient!

⚠️ Important: With 1 million employees and only 10 matching: IEnumerable transfers 1,000,000 rows then filters in memory. IQueryable transfers only 10 rows. Always use IQueryable with Entity Framework.

6. Interview Questions

Q: What is deferred execution in LINQ?

🎯 Interview Answer: A LINQ query is not executed when it is defined — it only executes when you iterate over the results. This means if the underlying collection changes between defining and iterating, the changes are reflected. Calling ToList() or ToArray() forces immediate execution and stores a snapshot preventing re-execution on every iteration.

Q: What is the difference between IEnumerable and IQueryable?

🎯 Interview Answer: IEnumerable executes filtering in C# memory — it pulls all data from the database first then filters. IQueryable translates LINQ to SQL and filters at the database level — only matching rows are transferred. For large datasets IQueryable is dramatically more efficient. Always use IQueryable when working with Entity Framework.
