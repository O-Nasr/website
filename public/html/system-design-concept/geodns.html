What Is GeoDNS?

Resolve a domain to different IPs based on where the user is located — before the HTTP request even happens.

Definition

GeoDNS (Geo-Routed DNS) is a DNS-based routing technique that resolves a domain name to different IP addresses based on the geographic location of the user. When a user requests example.com, GeoDNS looks at where the user is located and returns the IP address of the nearest or most appropriate data center.

🧠This happens before the actual HTTP request — during the DNS resolution phase. The user's browser never knows it was routed geographically.
2

How GeoDNS Works

5-step DNS resolution with geographic intelligence baked in.

The 5 Steps

1. User types example.com in browser. 2. DNS resolver sends query to authoritative DNS server. 3. GeoDNS checks the IP address of the requester (GeoIP lookup). 4. Based on location, returns the closest data center's IP. 5. Browser connects to that IP — faster response, lower latency.

Geographic Routing

Same domain, different IP — based on where you are in the world.

GeoDNS vs Load Balancer

GeoDNS routes between data centers. Load balancers distribute within a data center.

Interview Questions

▶What is GeoDNS and when would you use it?
GeoDNS is a DNS-based routing technique that returns different IP addresses based on the requester's geographic location. Use it when you have multiple data centers globally and want users to automatically connect to the nearest one for lower latency, better performance, and fault tolerance.
▶How does GeoDNS differ from a Load Balancer?
GeoDNS operates at the DNS level (before any HTTP connection) and routes based on geographic location — it picks the right data center. A Load Balancer operates at the network/application level (after connection) and distributes based on server health and load — it picks the right server within a data center. They work together in production.
▶What's the difference between GeoDNS and Anycast?
Both route users to nearby servers, but at different layers. GeoDNS resolves a domain to different IPs based on GeoIP lookup during DNS resolution. Anycast uses the same IP address everywhere, but the network's BGP routing automatically directs packets to the nearest instance. Anycast is common for DNS servers themselves and CDN edge nodes.
▶What happens if a GeoDNS data center goes down?
GeoDNS can be configured with health checks. If a data center is unhealthy, GeoDNS returns the IP of the next nearest healthy data center instead. This provides automatic failover at the DNS level — users are rerouted transparently.
