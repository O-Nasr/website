Consistent Hashing

A distributed hashing technique that maps keys to servers on a hash ring — minimizing data movement when nodes are added or removed.

The ProblemHash RingAdd/Remove NodesVirtual NodesComparison

The Problem with Modulo Hashing

hash(key) % N works until N changes — then almost all keys get remapped.
2

The Hash Ring

Keys and servers are placed on a circular ring. Each key maps to the next server clockwise.
🧠How it works: Keys and servers are hashed onto a circular ring (0 to 2³²). Each key walks clockwise until it reaches the next server — that server owns the key.
3

Adding & Removing Nodes

Only the keys between the new/removed node and its neighbor move — everything else stays.
✅Key insight: Adding or removing a node only moves ~K/N keys (where K = total keys, N = total nodes). With modulo hashing, ~80% of keys would move. This is why consistent hashing enables efficient, stable scaling.
4

Virtual Nodes

With only 3 physical nodes, keys may distribute unevenly. Virtual nodes solve this.
🧠Virtual nodes: Each physical server gets multiple points on the ring (e.g. A0, A1, A2...). More virtual nodes = more even key distribution. A server with more capacity can get more virtual nodes.

Interview Questions

▶Why can't we just use hash(key) % N in distributed systems?
Because when N changes (add/remove a server), almost all keys get remapped to different servers. This causes massive cache misses and data migration — making scaling extremely expensive.
▶How does consistent hashing minimize data movement?
Keys and servers are placed on a ring. Each key belongs to the next server clockwise. When a server is added/removed, only the keys between that server and its neighbor are affected — everything else stays. On average, only K/N keys move (where K = total keys, N = total servers).
▶What problem do virtual nodes solve?
With few physical nodes, keys may cluster unevenly — one server could own most of the ring. Virtual nodes spread each server across multiple points on the ring, ensuring even distribution. More virtual nodes = more even load. Servers with more capacity can get more virtual nodes.
▶Name systems that use consistent hashing.
Distributed caches (Memcached, Redis clusters), distributed databases (Cassandra, DynamoDB), load balancers, CDNs for edge server routing, and Kafka for partition assignment.
