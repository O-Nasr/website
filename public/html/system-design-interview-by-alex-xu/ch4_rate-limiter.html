Rate Limiting Algorithms

Four algorithms for controlling request rates — Fixed Window Counter, Sliding Log, Sliding Window Counter, and Token Bucket.

Fixed WindowSliding LogSliding WindowToken BucketComparison
1

Fixed Window Counter

Count requests in fixed time intervals. Reset counter at each window boundary.

✅ Pros

  • Simple to implement
  • Predictable reset — easy to reason about
  • Low memory — just one counter per window

❌ Cons

  • Allows bursts at window boundaries
  • Not smooth — all capacity can be used at window edges
📌Use Case: Simple API rate limiting — allow 100 requests per minute per user. Common in internal APIs or low-traffic services where burstiness is acceptable.
2

Sliding Log

Store the timestamp of every request. Count how many fall within the last window.

✅ Pros

  • Most accurate — no boundary burst problem
  • Exact count within any rolling window

❌ Cons

  • High memory — stores every timestamp
  • Expensive for high-traffic services
📌Use Case: Payment gateways and financial APIs where exceeding limits could cause fraud. Stores exact timestamps for precise enforcement.
3

Sliding Window Counter

Combines fixed window simplicity with sliding window smoothness using weighted averages.

✅ Pros

  • Smooth rate limiting — no boundary bursts
  • Low memory — only 2 counters per window

❌ Cons

  • Slightly complex formula
  • Approximate — not exact count (close enough for most use cases)
📌Use Case: Social media APIs (Twitter, Facebook). Prevents bursts while allowing flexibility. The approximation is good enough for non-critical rate limiting.
4

Token Bucket

Tokens are added at a fixed rate. Each request consumes one token. No tokens = request denied.

✅ Pros

  • Allows controlled bursts — critical for buffering/streaming
  • Smooth overall rate control
  • Flexible — adjust bucket size and refill rate independently

❌ Cons

  • More complex to implement than fixed window
  • Two parameters to tune (bucket size + refill rate)
📌Use Case: Video streaming platforms (Netflix, YouTube). Allows bursts for buffering but enforces overall rate. Also used by AWS API Gateway and Stripe.
