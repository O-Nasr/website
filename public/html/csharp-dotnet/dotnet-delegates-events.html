What is a Delegate?

A type-safe reference to a method — pass methods as parameters, store them, call them later.
🧠int x = 5; holds an integer. string s = "Hi"; holds a string. MyDelegate d = SomeMethod; holds a METHOD. That's a delegate.
Raw Delegate — The Old Way
// Step 1 — declare the delegate type
public delegate void PrintDelegate(string message);

// Step 2 — create a method matching the signature
static void PrintMessage(string message)
{
    Console.WriteLine("Message: " + message);
}

// Step 3 — assign and call
PrintDelegate del = PrintMessage;
del("Hello!"); // calls PrintMessage("Hello!")
2

Action and Func — The Modern Way

Built-in delegate types so you don't declare your own every time.
Action (void) and Func (returns value)
// Action — delegate that returns void
Action<string> print = message => Console.WriteLine(message);
print("Hello!");

// Func — delegate that returns a value (last type = return type)
Func<int, int, int> add = (a, b) => a + b;
int result = add(3, 5); // returns 8

// Func with no parameters
Func<string> getName = () => "Omar";
Console.WriteLine(getName()); // Omar

Passing methods as parameters
static void ProcessData(List<int> numbers, Func<int, bool> filter)
{
    foreach (int n in numbers)
    {
        if (filter(n))
            Console.WriteLine(n);
    }
}

var numbers = new List<int> { 1, 2, 3, 4, 5, 6 };
ProcessData(numbers, n => n > 3);       // prints 4, 5, 6
ProcessData(numbers, n => n % 2 == 0);  // prints 2, 4, 6

Multicast Delegate

One call, many methods — all registered methods execute in sequence.
One delegate, three handlers
Action<string> del = null;
del += message => Console.WriteLine("Logger 1: " + message);
del += message => Console.WriteLine("Logger 2: " + message);
del += message => Console.WriteLine("Logger 3: " + message);

del("Error occurred!");
// Logger 1: Error occurred!
// Logger 2: Error occurred!
// Logger 3: Error occurred!

Events — The Safe Wrapper

An event wraps a delegate to enforce the publisher-subscriber pattern safely.
⚠️The event keyword restricts outside code to only subscribe (+=) or unsubscribe (-=). Only the class that owns the event can fire it.
❌ Raw delegate — dangerous
class Button
{
    public Action OnClick; // raw!
}

button.OnClick = null;     // ❌ wipes ALL
button.OnClick("hacked");  // ❌ fires from outside
button.OnClick += MyHandler; // ✅ fine
✅ Event keyword — safe
class Button
{
    public event Action OnClick; // safe!
}

button.OnClick = null;      // ❌ COMPILE ERROR
button.OnClick("hacked");   // ❌ COMPILE ERROR
button.OnClick += MyHandler; // ✅ only +=/-=
Full Publisher-Subscriber Example
class OrderService
{
    public event Action<string> OnOrderPlaced; // declare event

    public void PlaceOrder(string orderNumber)
    {
        Console.WriteLine("Order placed: " + orderNumber);
        OnOrderPlaced?.Invoke(orderNumber); // fire event safely
    }
}

class EmailService
{
    public void SendConfirmationEmail(string num)
        => Console.WriteLine("Email sent for: " + num);
}

class SmsService
{
    public void SendSms(string num)
        => Console.WriteLine("SMS sent for: " + num);
}

// Wire everything together
var orderService = new OrderService();
var emailService = new EmailService();
var smsService   = new SmsService();

orderService.OnOrderPlaced += emailService.SendConfirmationEmail;
orderService.OnOrderPlaced += smsService.SendSms;

orderService.PlaceOrder("ORD-001");
// Order placed: ORD-001
// Email sent for: ORD-001
// SMS sent for: ORD-001
✅OrderService has no idea that EmailService or SmsService exist. It just fires the event. This is loose coupling.

Interview Questions

▶What is a delegate in C#?
A delegate is a type-safe reference to a method that allows methods to be passed as parameters, stored in variables, and called later. Action and Func are built-in delegate types provided by .NET as shortcuts. Multicast delegates can hold references to multiple methods and call them all in sequence with a single invocation.
▶What is the difference between a delegate and an event?
An event is a wrapper around a delegate that implements the publisher-subscriber pattern. While a raw delegate allows anyone to fire it or clear all subscriptions from outside the class, the event keyword restricts outside code to only subscribing or unsubscribing. Only the class that owns the event can fire it. This creates loose coupling where publishers and subscribers don't need to know about each other.
▶What is the publisher-subscriber pattern?
The publisher-subscriber pattern allows a class to notify other classes when something happens without knowing anything about them. The publisher defines an event and fires it when something occurs. Subscribers register their methods with the event using +=. When the event fires, all registered methods are called automatically. This promotes loose coupling between classes.
