What is LINQ?

LINQ provides a unified way to query collections, databases, and other data sources using method syntax or query syntax — both produce identical results.

C#// Method syntax — preferred
var result = numbers.Where(n => n > 3).Select(n => n * 2).ToList();

// Query syntax — same result
var result = (from n in numbers where n > 3 select n * 2).ToList();

Deferred Execution — Most Important Concept

A LINQ query does NOT execute when defined. It only executes when you iterate. The query is a recipe — writing it doesn't cook the food.

⚠️ Danger #1: If the list changes after defining the query but before iterating, the changes ARE reflected.
⚠️ Danger #2: Iterating multiple times = multiple executions. With EF, that means multiple SQL queries!

Test Yourself

Q1: When does a LINQ query with .Where() execute?

.Where() is deferred — it builds a query plan but doesn't execute. Execution happens when you iterate (foreach) or call a materializing method (ToList, Count, First, etc.).

Q2: You define a query, then .Add(10) to the source list, then iterate. Is 10 included?

Because the query is deferred, it doesn't snapshot the data. When you finally iterate, it runs against the current state of the list — including the added element. Use .ToList() to capture a snapshot.

Q3: What's the key difference between IEnumerable and IQueryable?

IQueryable translates LINQ expressions into SQL, so filtering happens at the database — only matching rows transfer. IEnumerable pulls ALL data first, then filters in C# memory. For large datasets, IQueryable is dramatically more efficient.

Q4: You call .Count() then foreach on a deferred query against Entity Framework. How many SQL queries fire?

Each enumeration of a deferred query is a separate execution. .Count() iterates once (query #1), foreach iterates again (query #2). Fix: call .ToList() first, then work with the materialized list.
