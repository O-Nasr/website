Value Types vs Reference Types and Null

C#int    age  = null;     // ❌ compile error — value types cannot be null
string name = null;     // ✅ fine — reference types can be null

// Stack — value types: int, bool, double, char → always has a value
// Heap  — reference types: string, class, array → can point to null
2

Nullable Value Types with ?

C#int?      age      = null; // ✅ nullable int
bool?     isActive = null; // ✅ nullable bool
double?   salary   = null; // ✅ nullable double
DateTime? dob      = null; // ✅ nullable DateTime

// int? is shorthand for Nullable<int>
int? age = 25;
Console.WriteLine(age.HasValue); // true
Console.WriteLine(age.Value);    // 25

int? salary = null;
Console.WriteLine(salary.HasValue); // false
Console.WriteLine(salary.Value);    // ❌ InvalidOperationException!
⚠️ Always check .HasValue before accessing .Value — accessing Value on a null Nullable throws InvalidOperationException.

? — Nullable Type

Makes value types nullable.

int? age = null; // shorthand for Nullable<int>

?? — Null Coalescing

Returns left side if not null, otherwise returns right side (fallback).

string city = null;
string result = city ?? "Cairo"; // "Cairo"

// Chaining — first non-null wins
string display = firstName ?? lastName ?? username ?? "Guest";

??= — Null Coalescing Assignment (C# 8+)

Assigns ONLY if left side is null.

string name = null;
name ??= "Default"; // name was null → assigned "Default"

string city = "Cairo";
city ??= "Alexandria"; // city NOT null → unchanged → "Cairo"

?. — Null Conditional Operator

Safe member access — returns null instead of NullReferenceException.

string name = null;
int? length = name?.Length; // null instead of crash ✅

// Chaining — multiple levels deep
string city = order?.Customer?.Address?.City;
// If ANY link is null → returns null safely

// Best pattern — combine ?. with ??
string city = order?.Customer?.Address?.City ?? "Unknown";

! — Null Forgiving Operator (C# 8+)

Tells compiler "trust me, this is not null." Suppresses nullable warnings. Use sparingly.

string? name = GetName(); // might return null
int length = name!.Length; // ! suppresses warning — you take responsibility

Nullable Reference Types (C# 8+)

C#// Enable in .csproj
<Nullable>enable</Nullable>

// After enabling — compiler tracks nullability
string  name = null;  // ⚠️ WARNING — non-nullable shouldn't be null
string? city = null;  // ✅ explicitly marked as nullable

// Compiler warns on unsafe access
string? name = GetName();
Console.WriteLine(name.Length); // ⚠️ WARNING — name might be null

// Fix options:
if (name != null) Console.WriteLine(name.Length);  // null check
Console.WriteLine(name?.Length);                   // null conditional
Console.WriteLine(name!.Length);                   // null forgiving

Guard Clauses — Null Checking at Entry

C#public void ProcessOrder(Order order)
{
    // Classic guard clause
    if (order == null)
        throw new ArgumentNullException(nameof(order));

    // Modern C# 10+ way
    ArgumentNullException.ThrowIfNull(order);

    // Safe to use order here
    Console.WriteLine(order.OrderNumber);
}
6

Test Yourself

Q1: What does city ?? "Cairo" return if city is null?

The ?? operator returns the left side if not null, otherwise the right side. Since city is null, it returns the fallback "Cairo".

Q2: What does order?.Customer?.City do if Customer is null?

The ?. operator short-circuits: if ANY link in the chain is null, it immediately returns null instead of throwing. That's why it's called the "null conditional" operator.

Q3: What's the difference between ? on value types vs reference types?

For value types, ? creates a Nullable<T> struct with HasValue and Value properties. For reference types (C# 8+ NRTs), it tells the compiler to track nullability and warn on unsafe access — it's purely a compiler hint, no runtime struct.

Q4: When does name ??= "Default" assign?

??= is the null coalescing assignment. It assigns the right side ONLY if the left side is currently null. If name already has a value, it stays unchanged.
