List, Dictionary, HashSet, Queue, Stack, LinkedList — performance characteristics, internal mechanics, and when to use each.

List<T> — Ordered Resizable Array

Backed by an internal array. When full, doubles in capacity (amortized growth).
List<T> — performance
List<int> list = new List<int> { 1, 2, 3, 4, 5 };

int item = list[2];          // O(1) — direct index access — fast ✅
bool ex  = list.Contains(4); // O(n) — scans every element — slow 😐
list.Add(6);                // O(1) amortized — fast ✅
list.Insert(2, 99);         // O(n) — shifts all after index — slow 😐
list.Remove(3);              // O(n) — shifts all after removed — slow 😐
📌Use for: ordered collection, index access, duplicates OK. Avoid for: frequent Contains (→ HashSet) or middle insert/remove (→ LinkedList).
2

Dictionary<TKey, TValue> — Hash Map

Uses hashing for O(1) key lookups. GetHashCode() → bucket index → direct jump.
Dictionary — hashing internals
// Adding "Ali" → 25
// Step 1: hash  → "Ali".GetHashCode() → 12345678
// Step 2: bucket → 12345678 % bucketCount → bucket 3
// Step 3: store ("Ali", 25) in bucket 3
//
// Reading dict["Ali"]
// Step 1: hash → 12345678
// Step 2: jump to bucket 3
// Step 3: return 25
// No scanning. No looping. Direct jump. O(1). ✅
Dictionary — performance + TryGetValue
Dictionary<string, int> dict = new();
dict["Ali"] = 25;                             // O(1) add
int age = dict["Ali"];                          // O(1) read ✅
bool ex = dict.ContainsKey("Ali");              // O(1) ✅
bool ev = dict.ContainsValue(25);               // O(n) — scans all 😐

// ✅ Safe — no exception
if (dict.TryGetValue("Ali", out int val))
    Console.WriteLine(val);

// Modern iteration
foreach (var (key, value) in dict)
    Console.WriteLine($"{key}: {value}");
3

HashSet<T> — Unique Values, Fast Lookup

Dictionary with only keys. O(1) Contains. Duplicates silently ignored.
HashSet + Set Operations
HashSet<int> ids = new() { 1, 2, 3 };
ids.Add(1); // silently ignored — already exists
ids.Add(4); // added ✅
// Count = 4, not 5

// List Contains: O(n) — scans 1M elements 😱
// HashSet Contains: O(1) — instant hash lookup ✅

HashSet<int> a = new() { 1,2,3,4,5 };
HashSet<int> b = new() { 3,4,5,6,7 };
a.UnionWith(b);      // 1,2,3,4,5,6,7
a.IntersectWith(b);  // 3,4,5
a.ExceptWith(b);     // 1,2
4

Queue<T> — First In First Out (FIFO)

Queue — FIFO
Queue<string> q = new();
q.Enqueue("Doc1.pdf");   // add to BACK
q.Enqueue("Doc2.pdf");
q.Enqueue("Doc3.pdf");    // [Doc1 ← front ... Doc3 ← back]
string next = q.Dequeue(); // "Doc1.pdf" — removed from FRONT
string peek = q.Peek();    // "Doc2.pdf" — see front, no remove
📌Use for: print queues, message queues, task scheduling, BFS graph traversal.
5

Stack<T> — Last In First Out (LIFO)

Stack — LIFO
Stack<string> history = new();
history.Push("google.com");
history.Push("github.com");
history.Push("stackoverflow.com"); // ← TOP
string last = history.Pop();       // "stackoverflow.com" — removed
string curr = history.Peek();      // "github.com" — TOP now
📌Use for: browser back button, undo/redo, call stack, DFS, expression parsing.
6

LinkedList<T> — Fast Middle Insert/Remove

LinkedList — node-based
// Structure: null ← [1] ↔ [2] ↔ [3] → null
LinkedList<int> linked = new();
linked.AddFirst(1); linked.AddLast(2); linked.AddLast(3);

LinkedListNode<int> node = linked.Find(2);
linked.AddAfter(node, 99);  // [1, 2, 99, 3] — O(1) insert!
linked.Remove(node);          // [1, 99, 3] — O(1) remove!

Thread-Safe Collections

⚠️Standard collections are NOT thread-safe. Multiple threads = data corruption. Use System.Collections.Concurrent.
Concurrent collections
// ✅ Thread-safe Dictionary
ConcurrentDictionary<string, int> dict = new();
dict.TryAdd("key", 1);
dict.TryGetValue("key", out int val);
dict.AddOrUpdate("key", 1, (k, v) => v + 1);

Interview Questions

▶What is the difference between List and Dictionary?
List<T> is an ordered collection backed by an array — O(1) index access but O(n) search by value. Dictionary<K,V> uses hashing for O(1) lookup by key. Use List when order matters, Dictionary when fast key-based lookup is the priority.
▶What is the difference between List and HashSet?
List allows duplicates, maintains order, Contains is O(n). HashSet stores only unique values using hashing — Contains is O(1). For large datasets where you only need existence checking, HashSet is dramatically more efficient.
▶When would you use Queue vs Stack?
Queue is FIFO — first in, first out. Use for task queues, message processing, BFS. Stack is LIFO — last in, first out. Use for undo/redo, browser history, call stack simulation, DFS.
▶How does Dictionary achieve O(1) lookup?
Dictionary uses hashing — calls GetHashCode() on the key, maps the hash to a bucket index, and jumps directly to the correct bucket without scanning. Hash collisions are handled with chaining and are rare in practice.
▶Why are standard collections not thread-safe?
Standard collections aren't synchronized — simultaneous reads and writes can corrupt internal state (array resizing, bucket reorganization). Use ConcurrentDictionary, ConcurrentQueue, etc. from System.Collections.Concurrent for safe concurrent access.
