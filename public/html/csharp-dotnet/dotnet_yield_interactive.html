What is yield?

Instead of collecting everything first, yield returns ONE item at a time, on demand. Control bounces back and forth between caller and method.

C#// Without yield — collects everything first
static IEnumerable<int> FilterWithout()
{
    List<int> temp = new List<int>(); // ❌ extra temp collection
    foreach (int i in MyList)
        if (i > 3) temp.Add(i);
    return temp; // returns everything at once
}

// With yield — returns one item at a time
static IEnumerable<int> FilterWith()
{
    foreach (int i in MyList)
        if (i > 3) yield return i; // return ONE → pause → wait
}
⚠️ Think of yield like a ping pong game — control bounces between caller and method. The method remembers its position between each bounce.

Stateful Iteration

yield preserves local variables between calls. Running totals, counters — state persists across iterations.

C#static IEnumerable<int> RunningTotal()
{
    int runningTotal = 0; // STAYS ALIVE between yields

    foreach (int i in MyList)
    {
        runningTotal += i;
        yield return runningTotal; // send total → pause
    }
}
// Output: 1, 3, 6, 10, 15
// Normally method return destroys locals — yield PAUSES, not ends
Normally when a method returns — all local variables are destroyed. With yield, the method is paused, not ended — variables stay alive because the compiler generates a state machine class.
4

yield return vs yield break

C#yield return i;  // return this value and PAUSE — continue on next iteration
yield break;     // STOP iteration completely — like a return statement

static IEnumerable<int> StopAtThree()
{
    foreach (int i in MyList)
    {
        if (i > 3) yield break; // stop everything here
        yield return i;
    }
}
// Output: 1, 2, 3 — stops before 4

Test Yourself

Q1: What does yield return do?

yield return sends the current value to the caller and pauses execution. All local variables are preserved. When the caller asks for the next item, execution resumes exactly where it left off.

Q2: Why can local variables survive between yield return calls?

The compiler transforms any yield method into a state machine class implementing IEnumerator. Local variables become fields in this class, so they persist between calls. The method is paused, not ended.

Q3: What does yield break do?

yield break terminates the iterator. No more values are sent to the caller. It's like a return statement for iterator methods.

Q4: If the caller breaks out of foreach after 2 items, do the remaining items get processed?

yield is inherently lazy. The method only runs as far as the caller needs. If the caller breaks after 2 items, the method stops — remaining items are never processed. This is a key performance benefit.
