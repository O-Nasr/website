IS Keyword — Type Checking

IS simply asks a yes/no question: "Is this object of this type?"

C#object obj = "Hello";

if (obj is string)
{
    Console.WriteLine("Yes! obj is a string"); // ✅ prints this
}

// Works with inheritance too
Dog d = new Dog();
Console.WriteLine(d is Dog);    // true
Console.WriteLine(d is Animal); // true — Dog inherits Animal
Console.WriteLine(d is string); // false
3

AS Keyword — Safe Type Conversion

AS tries to convert. If it fails, returns null instead of throwing an exception.

C#// Successful conversion
object obj = "Hello";
string str = obj as string;
Console.WriteLine(str); // "Hello" ✅

// Failed conversion — returns null, NO crash
object obj = 42;
string str = obj as string;
Console.WriteLine(str == null); // true — safe ✅

// vs direct cast — DANGEROUS
object obj = 42;
string s = (string)obj; // 💥 InvalidCastException!
⚠️ AS never throws on failed conversion. Direct cast (string)obj throws InvalidCastException. Always prefer AS for uncertain conversions.
4

AS Only Works With Reference Types

C#object obj = 42;

int i = obj as int;    // ❌ COMPILE ERROR — AS doesn't work with value types

// Fix — use IS with pattern matching
if (obj is int i)      // ✅ pattern matching works fine
{
    Console.WriteLine(i);
}
AS returns null on failure, but value types like int and bool cannot be null. That's why AS only works with reference types.
5

Modern C# — Pattern Matching (C# 7+)

Combine IS with variable assignment in one line. The preferred modern approach.

Old Way (Two Steps)

if (obj is string)
{
    string s = (string)obj; // two ops
    Console.WriteLine(s);
}
→

Modern (One Shot)

if (obj is string s) // check + assign
{
    Console.WriteLine(s);
    // s is already typed!
}
C#// Switch pattern matching — very powerful
switch (obj)
{
    case string s:
        Console.WriteLine("String: " + s);
        break;
    case int i:
        Console.WriteLine("Int: " + i);
        break;
    case null:
        Console.WriteLine("Null");
        break;
}
Pattern matching eliminates the double-check overhead: is string checks the type, and the variable s is immediately assigned — one operation instead of two.

Test Yourself

Q1: What does obj as string return if obj is an int?

AS returns null when the conversion fails — it never throws an exception. This is what makes it safer than a direct cast (string)obj, which would throw InvalidCastException.

Q2: Why can't you use as with value types like int?

AS returns null when conversion fails. But int, bool, double etc. are value types that cannot be null. There's no way to signal failure, so AS is restricted to reference types only.

Q3: What does pattern matching if (obj is string s) do?

Pattern matching combines the type check and variable assignment into a single statement. If obj is a string, the variable s is immediately available as a typed string — no separate cast needed.

Q4: Dog d = new Dog(); — what does d is Animal return?

is checks the full inheritance chain. Since Dog inherits from Animal, a Dog instance IS also an Animal. So d is Animal returns true, and d is Dog also returns true.

Interview Answer Bank

IS checks if an object is compatible with a type and returns true or false without conversion. AS attempts to convert — returns the converted object if successful, null if it fails. AS is safer than a direct cast because it never throws an exception on failure.
Pattern matching (C# 7) extends IS to combine type checking and variable assignment in one statement. Instead of checking is string then casting separately, if (obj is string s) checks the type and assigns the typed variable in one operation — eliminates the double-check overhead and produces cleaner code.
AS returns null when conversion fails, but value types like int and bool cannot be null. Therefore AS only works with reference types and nullable value types. For value types, use pattern matching with IS which handles the type check and conversion safely.
