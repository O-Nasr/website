Records vs Classes vs Structs

Reference vs value semantics, mutability, and equality

Class Struct Record Assignment Sim Comparison When to Use Quiz
1

Class — Mutable Reference Type

Variables hold a reference (pointer) — not the object. Equality compares references by default.

C#var p1 = new PersonClass { Name = "Omar", Age = 25 };
var p2 = p1; // p2 is NOT a copy — same reference

p2.Name = "Ali"; // modifies the SAME object
Console.WriteLine(p1.Name); // "Ali" — p1 also changed! 😱

// Equality compares REFERENCES
var p3 = new PersonClass { Name = "Omar", Age = 25 };
var p4 = new PersonClass { Name = "Omar", Age = 25 };
Console.WriteLine(p3 == p4); // false — different objects in memory
2

Struct — Value Type

Assignment creates a complete independent copy. Equality compares values.

C#var s1 = new PersonStruct { Name = "Omar", Age = 25 };
var s2 = s1; // full copy — independent

s2.Name = "Ali"; // modifies s2 ONLY
Console.WriteLine(s1.Name); // "Omar" — unchanged ✅

// Equality compares VALUES
var s3 = new PersonStruct { Name = "Omar", Age = 25 };
var s4 = new PersonStruct { Name = "Omar", Age = 25 };
Console.WriteLine(s3 == s4); // true — same content ✅
⚠️ Microsoft guideline: Structs should be 16 bytes or less. Larger structs are expensive to copy — every assignment, method call, and return copies ALL fields.
3

Record — Immutable Reference Type (C# 9+)

Reference type like class BUT with value equality. Designed for immutable data. Compiler auto-generates equality, ToString, constructor, and with support.

C#// One line — compiler generates EVERYTHING
public record PersonRecord(string Name, int Age);

// Value equality
var r1 = new PersonRecord("Omar", 25);
var r2 = new PersonRecord("Omar", 25);
Console.WriteLine(r1 == r2); // ✅ TRUE — value equality!

// Immutable — init only
r1.Name = "Ali"; // ❌ COMPILE ERROR — init only!

// with expression — non-destructive mutation
var updated = r1 with { Age = 26 }; // new record — original untouched

// Auto-generated ToString
Console.WriteLine(r1); // PersonRecord { Name = Omar, Age = 25 }
Record generates for free: value equality (==, Equals, GetHashCode), ToString, with-expression support, constructor, init-only properties.

When to Use Which

CLASS Use When

Complex behavior & methods. Mutable state that changes over time. Need inheritance. Examples: OrderService, UserRepository, HttpClient

STRUCT Use When

Small lightweight data (2-4 fields). Short-lived, created in bulk. ≤16 bytes. Examples: Point(X,Y), Color(R,G,B), DateTime, Guid

RECORD Use When

Immutable data. Equality based on content. DTOs & value objects. Examples: OrderDto, AddressValue, ApiResponse, SearchQuery

The key question: Should two objects with the same data be considered equal? If yes → record. If no → class. If small value type → struct.
7

Test Yourself

Q1: var p2 = p1; where p1 is a class instance. You change p2.Name. What happens to p1?

Classes are reference types. var p2 = p1 copies the reference (pointer), not the object. Both p1 and p2 point to the same object in memory, so changing one affects the other.

Q2: Two PersonRecord instances with identical data — are they equal?

Records have auto-generated value equality. Two records with the same property values are considered equal, even though they're different objects on the heap. This is the biggest difference from classes.

Q3: What does the with expression do on a record?

Since records are immutable, with creates a brand new record based on the original with specific properties changed. The original is never modified. This is called non-destructive mutation.

Q4: Why should structs be kept ≤16 bytes?

Structs are value types — every assignment, method call, and return creates a complete copy of all fields. A small 2-field struct copies cheaply. A large struct with many fields copies everything every time, wasting CPU and memory.

Interview Answer Bank

Classes are reference types on the heap — assignment copies the reference, so multiple variables point to the same object. Structs are value types on the stack — assignment creates a complete copy. Classes use reference equality; structs use value equality.
A record (C# 9) is a reference type like a class but with value equality. Compiler auto-generates equality, GetHashCode, ToString, and with-expression support. Uses init-only properties for immutability. Ideal for DTOs, API models, and value objects.
Creates a new record based on an existing one with specific properties changed — the original is not modified. Non-destructive mutation: var updated = original with { Age = 26 } creates a new record identical to original except Age is 26.
Use record when equality should be based on content — DTOs, value objects, configuration. Use class when the object has mutable state or complex behavior. Key question: should two objects with the same data be equal? Yes → record. No → class.
Structs are value types — every assignment, call, and return copies all fields. Small structs (2-4 fields) copy cheaply. Large structs copy everything every time, consuming CPU and memory. Microsoft recommends ≤16 bytes. If larger — use a class where only the reference is copied.
