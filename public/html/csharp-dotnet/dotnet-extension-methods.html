What Are Extension Methods?

Static method in a static class. First parameter uses "this" keyword. Three rules.
Extension method — three rules
// Rule 1: Static class — required
public static class StringExtensions
{
    // Rule 2: Static method — required
    // Rule 3: "this string" = extends string type
    public static string Capitalize(this string input)
    {
        if (string.IsNullOrEmpty(input)) return input;
        return char.ToUpper(input[0]) + input.Substring(1).ToLower();
    }
}

// Usage — looks like a built-in string method!
string name = "omar";
Console.WriteLine(name.Capitalize()); // Omar

With Parameters

Additional parameters after "this"
public static string Repeat(this string input, int count)
    => string.Concat(Enumerable.Repeat(input, count));

public static bool ContainsAny(this string input, params string[] values)
    => values.Any(v => input.Contains(v));

Console.WriteLine("Ha".Repeat(3));                     // HaHaHa
bool r = "Hello World".ContainsAny("Hello", "Bye"); // true

Extending Any Type

int, DateTime, bool, object
public static bool IsBetween(this int val, int min, int max)
    => val >= min && val <= max;

public static bool IsWeekend(this DateTime date)
    => date.DayOfWeek == DayOfWeek.Saturday || date.DayOfWeek == DayOfWeek.Sunday;

public static string ToYesNo(this bool val) => val ? "Yes" : "No";

// Reads like natural language
int age = 25;
bool valid = age.IsBetween(18, 65);     // true
Console.WriteLine(true.ToYesNo());      // Yes

Extending Generic Types

IEnumerable<T> extensions
public static bool IsNullOrEmpty<T>(this IEnumerable<T> source)
    => source == null || !source.Any();

public static void AddIfNotExists<T>(this List<T> list, T item)
{
    if (!list.Contains(item)) list.Add(item);
}

List<int> numbers = null;
bool empty = numbers.IsNullOrEmpty(); // true — no NullReferenceException! ✅

Extending Interfaces — Most Powerful

Extend an interface → ALL implementations get the method for free.
Extend interface — all implementations get it free
public interface INotificationService
{
    void Send(string message);
}

public static class NotificationExtensions
{
    public static void SendWithTimestamp(
        this INotificationService svc, string msg)
    {
        svc.Send($"[{DateTime.Now:HH:mm:ss}] {msg}");
    }

    public static void SendUrgent(
        this INotificationService svc, string msg)
    {
        svc.Send($"🚨 URGENT: {msg}");
    }
}
// EmailNotification, SmsNotification, and ANY future
// implementation gets both methods for free! ✅
💡This is exactly how LINQ works. Where(), Select(), OrderBy() are all extension methods on IEnumerable<T>. Any class implementing IEnumerable<T> gets all LINQ methods automatically.

How LINQ is Built

LINQ = extension methods on IEnumerable<T>
// LINQ methods are ALL extension methods on IEnumerable<T>
var result = numbers
    .Where(n => n > 3)    // extension method
    .Select(n => n * 2)   // extension method
    .OrderBy(n => n)      // extension method
    .ToList();            // extension method
// Chaining works because each returns IEnumerable<T>

Real World Patterns

Fluent Validation + WhereIf
// Fluent Validation
public static string RequiredField(this string val, string name)
{
    if (string.IsNullOrWhiteSpace(val))
        throw new ArgumentException($"{name} is required");
    return val;
}

customerName.RequiredField("Customer Name"); // clean!

// Conditional Query — WhereIf
public static IQueryable<T> WhereIf<T>(
    this IQueryable<T> query, bool condition,
    Expression<Func<T, bool>> predicate)
    => condition ? query.Where(predicate) : query;

// Usage — conditionally apply filters → all SQL
return dbContext.Employees
    .WhereIf(!string.IsNullOrEmpty(name), e => e.Name.Contains(name))
    .WhereIf(minSalary.HasValue, e => e.Salary >= minSalary.Value);

Interview Questions

▶What are extension methods and how do you create them?
Extension methods allow adding new methods to existing types without modifying them. Defined as static methods in a static class where the first parameter uses the this keyword. They appear to callers as built-in methods but live in a separate class.
▶What is a real-world example?
LINQ is the most prominent example — Where, Select, OrderBy are all extension methods on IEnumerable<T>. Any class implementing IEnumerable<T> gets all LINQ functionality automatically. ASP.NET Core's AddScoped, UseMiddleware, MapControllers are also extension methods.
▶What are the limitations?
Cannot access private members. Instance methods always win over extensions with the same name. Namespace must be imported. Cannot override existing methods — silently ignored if class already has one.
▶Why is extending interfaces powerful?
Every class implementing the interface gets the new methods without code changes. Follows Open/Closed principle. This is exactly how LINQ works — IEnumerable<T> is extended with dozens of methods that work for List, Array, Dictionary, and any future collection type automatically.
