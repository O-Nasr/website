The Three Concepts

Click each card for a detailed example:

🔒

Consistency

All users see the same data at the same time, no matter which server.

🟢

Availability

System responds to every request with non-error, even during failures.

🛡️

Reliability

System keeps working correctly over time, even in face of failures.

Consistency — Data Correctness Across Systems

If you update a user's profile on Server A, then immediately read from Server B, you should see the updated data. C in CAP Theorem.

Banking app: Transfer $100 from Account A to B. Both balances must reflect the change immediately everywhere. If Server 2 still shows old balance → inconsistent.

Availability — System Responsiveness

Every request gets a non-error response, even if some components fail. May return slightly stale data rather than blocking. A in CAP Theorem.

Social media feed: During a server outage, you still see posts (maybe cached/slightly outdated). The app doesn't show an error screen. Usable even if consistency is temporarily sacrificed.

Reliability — Long-Term Correctness

System doesn't crash, doesn't lose data, responds as expected over time. About trustworthiness and fault tolerance. Broader concept than CAP.

Messaging app: Never loses messages, even during a server crash or network issue. Every message sent is eventually delivered correctly.

Test Yourself

Q1: User updates profile on Server A. Immediately reads from Server B and sees OLD data. Which property is violated?

Consistency requires all users to see the same data at the same time. Server B returning stale data means the system is inconsistent — even though it's available (responded) and reliable (didn't crash).

Q2: During a server outage, a social media app shows cached posts instead of an error. Which property is prioritized?

The app stays usable (available) by serving cached content, even though the data may be slightly stale (sacrificing consistency). This is the classic AP trade-off.

Q3: A messaging app never loses messages even during crashes. Which property does this demonstrate?

Reliability is about the system working correctly over time and not losing data — even during failures. It's the broader concept of trustworthiness and fault tolerance.

Q4: In a banking transfer, what does the CAP theorem force you to choose during a network partition?

During a network partition, you must choose: block the request until nodes synchronize (Consistency) or respond immediately with potentially stale data (Availability). Banking systems typically choose CP — correctness over speed.
