01 The Problem

Monolith: one log file, grep "error" app.log, done. Microservices: 10 servers, 10 log streams. User reports "my order failed" — which service caused it? Console.WriteLine: SSH into each server, try to match timestamps across 1000s of requests per second. Containers restart — logs are GONE.

03 Correlation ID + OpenTelemetry

Unique ID generated at Gateway, propagated via X-Correlation-Id header. Every log includes it. Search one ID → entire request journey.

public class CorrelationIdMiddleware
{
    public async Task InvokeAsync(HttpContext context)
    {
        var id = context.Request.Headers["X-Correlation-Id"]
                 .FirstOrDefault() ?? Guid.NewGuid().ToString();
        context.Response.Headers["X-Correlation-Id"] = id;
        using (LogContext.PushProperty("CorrelationId", id))
            await _next(context);
    }
}
builder.Services.AddOpenTelemetry()
    .WithTracing(t => t
        .AddAspNetCoreInstrumentation()       // auto-trace incoming
        .AddHttpClientInstrumentation()        // auto-trace outgoing
        .AddEntityFrameworkCoreInstrumentation()// auto-trace EF
        .AddJaegerExporter())                  // change = change backend
    .WithMetrics(m => m
        .AddAspNetCoreInstrumentation()
        .AddPrometheusExporter());

OTel replaces client libraries, not backends. Instrument once, export to any backend by changing one config line:

Before OTel: Jaeger SDK + Prometheus client + Serilog = 3 libraries
With OTel:   OpenTelemetry SDK → Jaeger / Prometheus / Seq

Switch Jaeger → Zipkin? Change one line. Code stays same.

04 Interview Q&A

Q: What is a Correlation ID?
Unique ID generated at entry point, propagated through every service. Every log includes it. Search one ID → complete request journey across all services. Without it, matching logs across 10 services at 1000 req/sec is impossible.
Q: What is OpenTelemetry?
Vendor-neutral standard for telemetry — traces, metrics, logs. Instrument once, export to any backend (Jaeger, Prometheus, Datadog) by changing config. Replaces client libraries, not backends. Industry standard, prevents vendor lock-in.
